# Decisões Técnicas

## FASE 0 — Setup e Fundação

| Decisão | Alternativas | Escolha | Motivo |
|---------|-------------|---------|--------|
| Framework RAG API | FastAPI, Flask, Django | FastAPI | Performance assíncrona, suporte nativo a OpenAPI, tipagem com Pydantic |
| Framework Legacy API | ASP.NET Core, Minimal API | Minimal API | Sintaxe moderna, menos boilerplate, boa para microsserviços |
| Framework Notification | Express, Fastify, NestJS | Express | Mais conhecido, amplo ecossistema, simples o suficiente |
| TypeScript no Notification | JS puro, TS | TypeScript | Tipagem estática, melhor manutenção, expectativa do mercado |
| Orquestração local | Docker Compose, Kubernetes | Docker Compose | Simplicidade, adequado para dev local e CI |
